1. modified, staged, commited.
2. команда add використовується для того щоб додати файли до диекторії.
3. команда status використовується для перегляду стану файлів в диекторії.
4. команда commit використовується для того щоб зафіксувати зміни.
5. команда init використовується для запуску всіх необхідних процесів.
6. в коментарях комітів необхідно писати що було змінено або додано.
7. команда checkout використовується для того щоб змінити поточну гілку.
8. команда reset використовується для скасування речей.
9. команда log використовується для того щоб показати досяжну записану історію проекту.
10.команда status використовується для того щоб показати стани файлів у робочій директорії.
11.відновити кілька файлів з попередньо створеного коміта можна за допомогою 
   команд git log, git reset --hard
12.перейди до попереднього коміта по імені можна за допомогою команди git log.
13.створити нову гілку можна за допомогою команди git branch і назва потрібної гілки.
14.перейти на нову гілку можна перейти за допомогою команди git checkout і назва потрібної гілки.
15.перейти на основну гілку можна перейти за допомогою команди git checkout і назва основної гілки.
16.виконати злиття гілок можна виконати будучи на потрібній гілці та використовуючи
   команду git merge і назва іншої потрібної гілки
